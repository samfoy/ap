### Step 7: `src/main.rs` — wiring

Both `run_headless` and `run_tui` currently have:
```rust
let middleware = shell_hook_bridge(&config.hooks);
```

Replace with:
```rust
let global_dir = config.skills.global_dir.clone()
    .unwrap_or_else(|| dirs::home_dir().unwrap_or_default().join(".ap").join("skills"));
let project_dir = config.skills.project_dir.clone()
    .unwrap_or_else(|| std::path::PathBuf::from(".ap/skills"));
let skill_dirs: Vec<std::path::PathBuf> = [global_dir, project_dir]
    .into_iter()
    .filter(|d| d.exists())
    .collect();
let loader = ap::skills::SkillLoader::new(skill_dirs);

let middleware = if config.skills.enabled {
    shell_hook_bridge(&config.hooks)
        .pre_turn(ap::skills::skill_injection_middleware(loader, config.skills.clone()))
} else {
    shell_hook_bridge(&config.hooks)
};
```

Note: `dirs::home_dir()` returns `Option<PathBuf>`. Use `.unwrap_or_default()` to get empty path on failure (skills dir won't exist → silently skipped by SkillLoader). This satisfies `clippy::unwrap_used` denial.

## Constraints

1. **Compile order matters**: Steps 1 and 2 must land before 3–7. Step 2 is atomically breaking — all 5 locations must be updated in one commit.
2. **Clippy deny**: `clippy::unwrap_used` and `clippy::expect_used` are denied in production code. Use `unwrap_or_default()`, `ok()`, `?` etc. Tests get `#[allow(...)]`.
3. **No new crates**: Pure Rust TF-IDF + stdlib I/O. All required crates are already in `Cargo.toml`.
4. **`#[serde(skip)]` vs `#[serde(default, skip_serializing_if)]`**: Use `#[serde(skip)]` (skip both directions) — simpler and matches FR-1 rationale.
5. **`MockProvider` in `turn.rs` tests**: Must add `_system_prompt: Option<&'a str>` to both `MockProvider` and `ErrorProvider` when the trait signature changes in Step 2.
6. **`dirs::home_dir()` returns `Option<PathBuf>`**: Avoid `.unwrap()` — use `.unwrap_or_default()` or `if let Some(h) = dirs::home_dir()` pattern.
7. **`AppConfig` derives `Default`**: Adding `pub skills: SkillsConfig` is safe as long as `SkillsConfig` implements `Default`.
