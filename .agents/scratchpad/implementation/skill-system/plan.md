### Integration Test (`tests/skill_injection.rs`)

Full end-to-end through `SkillLoader → select_skills → skill_injection_middleware` without any real LLM call.

**Scenario**: Multi-directory later-wins + TF-IDF selection

```
Setup:
  global/
    git.md      — body: "Use git to commit and push changes"
    shared.md   — body: "GLOBAL version"
  project/
    docker.md   — body: "Use docker to build and run containers"
    shared.md   — body: "PROJECT version"

Step 1: Build SkillLoader([global, project])
Step 2: Call .load() → assert 3 skills: git, docker, shared
Step 3: Assert shared.body == "PROJECT version" (later-wins)

Step 4: Build Conversation with one user message: "I need help with git commit"
Step 5: Call select_skills(skills, messages, 5)
Step 6: Assert git skill is in results, docker skill is NOT

Step 7: Call skill_injection_middleware(loader, config)(conv)
Step 8: Assert returns Some(conv), conv.system_prompt contains "git"

Step 9: Build empty Conversation (no messages)
Step 10: Call middleware → assert returns None
```

### E2E Test Scenario (Validator manual execution)

**Harness**: Real CLI execution with temp skill files

```bash
# Setup
mkdir -p /tmp/ap-skills-test
cat > /tmp/ap-skills-test/git-workflow.md << 'EOF'
---
tools: [bash]
---
## Git Workflow

Always use conventional commits: feat:, fix:, chore:
EOF

# Configure ap to use temp skills dir
export AP_SKILLS_DIR=/tmp/ap-skills-test

# Run ap with a git-related query (headless mode)
echo "How do I commit my changes using conventional commits?" | ap --headless

# Expected: system_prompt in Bedrock request contains "Git Workflow" section
# Adversarial path: remove the skill file mid-run to verify graceful degradation
rm /tmp/ap-skills-test/git-workflow.md
echo "Another git question" | ap --headless
# Expected: runs without crash, no system_prompt injection (file gone)
```
